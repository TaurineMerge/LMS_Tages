# Инструмент статического анализа кода (Linter)

Для обеспечения высокого качества, читаемости и отсутствия распространенных ошибок в коде, проект adminPanel использует статический анализатор кода `golangci-lint`.

## Что такое `golangci-lint`?
Это агрегирующий линтер для Go, который запускает множество других линтеров (таких как `staticcheck`, `govet`, `revive` и др.) за один проход. Это значительно ускоряет анализ и обеспечивает комплексную проверку.

## Зачем он нужен?
- **Повышение качества кода:** Обнаруживает потенциальные баги, неэффективный код и проблемы безопасности.
- **Единый стиль:** Обеспечивает соответствие кода общепринятым стандартам и стилю Go.
- **Раннее обнаружение ошибок:** Помогает найти проблемы до стадии компиляции или выполнения.
- **Улучшение читаемости:** Способствует написанию более чистого и понятного кода.

## Как установить?
Используйте следующую команду для установки `golangci-lint`:
```bash
go install github.com/golangci/golangci-lint/v2/cmd/golangci-lint@v2.7.2
```
[Cсылка](https://github.com/golangci/golangci-lint) на репозиторий.

## Как запустить?
Для запуска полного анализа всех файлов проекта:
```bash
golangci-lint run
```

## Как запустить автоисправление?
Некоторые ошибки линтера могут быть автоматически исправлены. Для этого используйте флаг `--fix`:
```bash
golangci-lint run --fix
```

## Конфигурация
Все правила и настройки линтера находятся в файле **`.golangci.yml`** в корне проекта `adminPanel`.
Этот файл определяет, какие линтеры включены, их специфические настройки и исключения. Он гарантирует, что все разработчики используют одинаковые правила проверки кода.

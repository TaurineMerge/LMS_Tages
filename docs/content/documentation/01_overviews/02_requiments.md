## Бизнес требования

1. При входе в систему должна осуществляться авторизация и аутентификация пользователя через Keycloak.
2. Гости (неавторизованные пользователи) должны иметь возможность просматривать категории, курсы и уроки без авторизации.
3. Для прохождения тестов и получения сертификатов пользователь должен быть авторизован.
4. Управление контентом в административной панели должно осуществляться в табличном виде с возможностью применения следующих инструментов:

* пагинация
* фильтрация
* сортировка

5. Статистика прохождения тестов в личном кабинете должна отображать следующие данные:

* количество попыток прохождения теста в рамках одного курса
* последний результат тестирования
* среднее количество баллов за все тесты
* общее количество пройденных тестов за все курсы
6. При вводе данных в формы (создание/редактирование категорий, курсов, уроков, тестов) должна осуществляться валидация.
7. Курсы, уроки и тесты должны иметь статусную модель (например, "черновик" и "опубликован") для управления их видимостью.
8. Взаимодействие между компонентами системы должно осуществляться по REST API.
9. Административная панель должна предоставлять редактор для создания и редактирования уроков с поддержкой:

* форматирования текста (жирный, курсив, подчеркивание, цвет)
* изменения стилей (заголовки, параграфы)
* изменения кегля и начертания шрифта
* вставки изображений
12. Административная панель должна позволять загружать изображения для курсов в S3-хранилище (MinIO) и управлять ими.
13. Интеграция со следующими системами:

* Keycloak для аутентификации и авторизации
* S3-совместимое хранилище (MinIO) для хранения изображений
* Модуль тестирования (конструктор тестов)
* Личный кабинет пользователя

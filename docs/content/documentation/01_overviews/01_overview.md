**LMS** - это система управления обучением, предназначенная для организации и проведения учебных процессов. Студенты могут знакомиться с курсами и уроками без обязательной регистрации, а после авторизации — проходить итоговое тестирование и получать сертификаты о завершении обучения. Преподавателям система предоставляет инструменты для создания собственных курсов, уроков и тестов, позволяя эффективно передавать знания и оценивать результаты учащихся.

### Основной функционал: 
> Функционал может меняться в зависимости от роли пользователя.

> Для студентов:

---


- Просмотр курсов и уроков без обязательной регистрации
- Прохождение итогового тестирования после авторизации
- Получение сертификатов о завершении обучения
- Отслеживание своего прогресса обучения

---
> Для преподавателей:
---

- Создание собственных курсов, уроков и тестов
- Подача заявок на публикацию материалов (согласование контента)
- Отслеживание статуса заявок на публикацию
- Внесение правок в материалы по замечаниям
- Просмотр заявок коллег для координации работы
- Совершение действий по согласованию контента: утверждение, отклонение, отправка на доработку, отзыв заявки
---
> Для администраторов:
---
- Управление пользователями и ролями
- Согласование заявок на публикацию учебных материалов
- Контроль качества контента
- Управление структурой курсов и категорий
- Система поддерживает полный цикл работы с учебными материалами — от создания и согласования контента преподавателями до обучения студентов и выдачи сертификатов.

### FE
Пользовательский интерфейс предоставляет возможность выполнять все перечисленные действия напрямую из приложения, обеспечивая удобство и прозрачность работы.

### BE
BE выполняет функции маршрутизации запросов с FE, обеспечивая бизнес-логику и взаимодействие с различными системами и сервисами, такими как:

- **Admin Panel** — отвечает за управление контентом (создание, редактирование, удаление) внутри публичной панели;
- **Public Side** — публичный сайт, на который заходит гость с целью ознакомления с досутпными материалами (контентом), а также последующими прохождением тестов и получения сертификата;
- **Testing system** — система, позваляющая создавать тесты админу (учителю) с целью проверки знаний пользователя. Также ее использует пользователь (ученик) для прохождения итогового тестирования и получения сертификата;
- **Personal Account** — личный кабинет пользователя, в котором отслеживается статистика прохождения тестов учеником. Также туда приходят и хранятся сертификаты после успешного прохождения учеником итогового курсового теста;
- **Keycloak** — система авторизации и аутентификации пользователей в рамках единой системы LMS;
- **Nginx** — маршрутизатор, который позволяет редиректить пользователя к нужному модулю с учетом его запроса;
- **Swagger UI** — это интерактивный веб-интерфейс для визуализации и тестирования API, который автоматически генерируется из спецификации OpenAPI (ранее Swagger);
- **Jaegger** — система трассировки, позволяющая в режиме реального времени оценить работу приложения и отследить все логи (при необходимости исправить);
- **S3** — внешний сервис, позволяющий преобразовать ссылку изображения в изображение (сохраняет изображение в бакет) с последующим использованием изображения в создании курсов, уроков и тестов;
- **Knowledge-base-db** — основная база данных для хранения информации о пользователях, заявках и процессах согласования, а также для формирования данных, отображаемых на FE;
- **Personal-account-db** — основная база данных для хранения информации о пользователях, заявках и процессах согласования, а также для формирования данных, отображаемых на FE;
- **Testing-system-db** — основная база данных для хранения информации о пользователях, заявках и процессах согласования, а также для формирования данных, отображаемых на FE;


### Назначение

* Обеспечить пользователям возможность изучать образовательные материалы без обязательной регистрации, с опциональным прохождением итогового тестирования и получением сертификатов после авторизации;
* Предоставить преподавателям инструменты для создания учебного контента, включая возможность правок, отслеживания текущего статуса контента;
* Создать полноценную систему управления обучением с разделением на публичную часть для студентов и административную панель для преподавателей и администраторов;
* Организовать управление полным циклом работы с учебными материалами — от создания контента до обучения студентов и выдачи сертификатов;
* Предоставить администраторам инструменты для контроля качества контента, управления пользователями, ролями и структурой учебных материалов.

### Цели

* Обеспечить возможность свободного доступа к образовательным материалам для всех пользователей без необходимости предварительной регистрации;
* Предоставить гибкую систему обучения с опциональной сертификацией, где пользователи сами выбирают формат взаимодействия с контентом;
* Предоставить администраторам инструменты для самостоятельного создания и редактирования учебного контента;
* Создать единую экосистему обучения, охватывающую все этапы — от создания контента администратором до обучения студентов и выдачи сертификатов;
* Обеспечить разделение интерфейсов системы для разных типов пользователей с сохранением целостности данных и процессов.

### Задачи

* Разработать публичную часть сайта для гостей с возможностью просмотра категорий, курсов и уроков без авторизации;
* Реализовать систему аутентификации через Keycloak для регистрации и входа пользователей;
* Создать административную панель для управления учебным контентом (категории, курсы, уроки);
* Реализовать едактор для создания и редактирования уроков с поддержкой форматирования текста и загрузкой изображений;
* Интегрировать в административную панель функционал загрузки изображений курсов в S3-совместимое хранилище (MinIO) с сохранением ссылок в базе данных;
* Создать отдельный модуль тестирования с возможностью создания, редактирования и прохождения тестов;
* Реализовать личный кабинет пользователя для отслеживания статистики прохождения тестов;
* Настроить разграничение доступа между публичной частью, личным кабинетом и административной панелью;
* Реализовать систему управления видимостью контента (черновики/опубликованные материалы) для курсов, уроков и тестов;
* Создать web API для взаимодействия между всеми компонентами системы и внешними сервисами.
